C++ & OpenGL Demonstration

During the first lockdown of 2020, I explored programming in C++ and OpenGL and I started this individual project to practice on these areas.

This project utilised the Programmable Render Pipeline of OpenGL which involves the features:

  • Vertex Colouring & Texturing
  • Overlaying & Interactive Graphics
  • Perspective & Orthographic Projections
  • Translation, Rotation, Scale Transformations
  • Transformations & Vertex Based Animations
  • Object Instancing
  • Lighting Effects
    • Directional
    • Point-Based
    • Colouring
    • Vertex Emissions
  • Window Scalability

Two shaders were created for this project. The first being the Vertex shader, which describes the characteristics of the vertices in respect to position, projection, colour, lighting, normal and so on. The other is the Fragment shader which considers how each pixel is rendered by using information from the Vertex shader and textures.

C++ was involved to house and manage the OpenGL code of the application and object orientated approaches of Abstraction and Reuse ware applied.

An example of this concept is the ‘Buffers’ class, which manages the ID, data and the shader variable to send the data to, for each of the required buffers to render a visual object.

As many procedures in OpenGL are particular and repetitive, ensuring complexity was concealed and allowing versatile use was adopted to aid the involvement of more advanced implementations and outcomes.
